The Logfather SIEM

The Plan: Develop a small‑footprint, containerized SIEM platform that provides clear visibility into workstation hardware integrity, enabling early detection of unauthorized changes and supporting secure, maintainable IT environments.

Many academic labs and small IT teams lack the resources for enterprise SIEM tools but still face risks from hardware tampering, configuration drift, and limited monitoring capabilities. Existing solutions are often too complex, too costly, or too infrastructure‑heavy. A lightweight, reproducible system is needed to track workstation hardware states, establish baselines, and identify deviations that may indicate security issues or operational problems.

Planned System Components

  • Workstation Agent: PowerShell‑based collector for CPU, RAM, disk, BIOS, GPU, OS, and network configuration data.
  • API Layer: Flask service for receiving, validating, and storing hardware reports.
  • Dashboard: Web interface showing baseline vs. current hardware states for each workstation.
  • Data Storage: SQLite database for structured, portable log retention.
  • Containerization: Docker Compose for consistent deployment and reproducibility.

Planned Methods

  • Use CIM queries to gather hardware data from Windows workstations.
  • Implement REST endpoints for log ingestion and retrieval.
  • Build grouping logic to identify first‑seen (baseline) and latest (current) hardware states.
  • Design a dual‑panel dashboard layout for clear comparison.

Expected Outcomes

The proposed system is expected to produce a functional prototype demonstrating that meaningful workstation integrity monitoring can be achieved without enterprise‑scale SIEM tools. The project will deliver a reproducible, modular platform capable of collecting hardware data, establishing baselines, and highlighting deviations that may indicate tampering or system drift. The architecture is intentionally designed for future enhancements, including Windows event log ingestion, remote command execution, and automated alerting, making it a scalable foundation for broader security monitoring needs.
